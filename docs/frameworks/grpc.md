# gRPC Support

EasyApi provides built-in support for gRPC services, allowing you to export gRPC service definitions and call gRPC endpoints directly from the IDE.

## Enabling gRPC Support

1. Open **Preferences(Settings)** > **Other Settings** > **EasyApi** > **gRPC** tab
2. Enable **gRPC support** to recognize gRPC service classes
3. Optionally enable **gRPC call** to invoke gRPC endpoints from the IDE

## Auto Detection

When **Auto Detect** is enabled, EasyApi will automatically find gRPC runtime artifacts in your project dependencies. This includes the gRPC stubs and protobuf-generated classes.

## Runtime Artifacts

If auto-detection doesn't find the required artifacts, you can manually specify the gRPC runtime JARs:

1. Click the **+** button next to **Runtime Artifacts**
2. Select the gRPC JARs from your project or system libraries
3. Typical artifacts needed:
   - `grpc-api.jar`
   - `grpc-stub.jar`
   - `grpc-protobuf.jar`
   - `protobuf-java.jar`

## How gRPC Services Are Recognized

EasyApi recognizes gRPC services by checking if a class extends `io.grpc.BindableService` (the standard gRPC service interface generated by protobuf).

You can customize this behavior with the `class.is.grpc` rule:

```properties
class.is.grpc=groovy:it.isExtend("io.grpc.BindableService")
```

## Exporting gRPC APIs

Once gRPC support is enabled, you can export gRPC service methods just like REST APIs:

- **Export to Postman**: gRPC methods will be included in the collection
- **Export to Markdown**: gRPC service documentation will be generated
- **Export to RPC**: gRPC services can be exported in RPC format

## Calling gRPC Endpoints

When **gRPC call** is enabled, you can invoke gRPC methods directly from the IDE:

1. Navigate to a gRPC service class
2. Use the EasyApi **Call** action on a method
3. Provide the request parameters in JSON format
4. View the response

## Configuration

### Custom gRPC Recognition

```properties
class.is.grpc=groovy:it.isExtend("io.grpc.BindableService") || it.hasAnn("io.grpc.stub.annotations.RpcMethod")
```

### gRPC Method Naming

```properties
api.name=groovy:it.ann("io.grpc.stub.annotations.RpcMethod")?.value() ?: it.name()
```

### Ignore Internal gRPC Methods

```properties
ignore=groovy:it.name() == "getServiceDescriptor" || it.name() == "bindService"
```
